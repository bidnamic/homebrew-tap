1.Install & upgrade

Bidnamic OS ships as a Homebrew package. Three commands the first time, one command to upgrade after that.

Before you install

  • macOS Tahoe (26.x) or later, on Apple Silicon
  • Tailscale Mac app — install, then sign in
  • Homebrew — if you don't already have it

Install

brew tap bidnamic/homebrew-tap
brew install bidnamic-os
bidnamic-os post-install

post-install is a one-time privileged step that registers the EFS mount helper and starts the watchdog. It will prompt for your macOS password.

First run

bidnamic-os

That's the command you'll type every time you want to talk to Brian. The first run pulls your cloud session up; it then drops you straight into the chat. Type hello and Brian will greet you with your portfolio count (see section 6, step 1).

If you ever want to re-open this tutorial:

bidnamic-os tutorial

Upgrade

bidnamic-os upgrade

Runs brew update, brew upgrade bidnamic-os, and re-runs post-install (so it'll prompt for your password again). New releases land within minutes of a launcher change — run this any time something feels off, or once a week as a habit.

Tip. If bidnamic-os ever hangs or refuses to connect, the usual fix is bidnamic-os upgrade followed by a fresh bidnamic-os. Beyond that, post in #delivery-team.

2.What this actually is

You'll open Claude Code in your terminal to talk to Brian — Bidnamic's assistant. Under the hood, Brian runs on top of Bidnamic OS: every playbook the team follows, every client's history, every healthcheck, every SQL query our analysts have written and tested.

Brian looks things up, investigates, explains, drafts, and proposes. You decide and act. Nothing goes into Admin UI unless you put it there yourself.

If you've poked at a general-purpose chatbot before, this will feel different. Answers here are grounded in real Bidnamic data and real procedures. When Brian tells you BobsWidgetStore's ROAS is tracking 18% below goal, it's because he just ran the query — not because he's guessing.

Brian's remit is strictly ecommerce marketing — campaigns, feeds, ROAS, Shopping, Search, PMax, Bing, attribution, lifecycle, reporting. Ask him to book a dentist appointment and he'll politely decline.

3.Four habits that make it work

Two minutes to learn. Saves you hours later. These apply to any AI tool you'll use at Bidnamic, not just this one.

Pick the right job for it

Most of the time you'll just type plain English and let it figure out what you want. That's the expected way in — talk to it like you'd brief a smart junior colleague. Slash commands like /client:portfolio-review exist as shortcuts for when you already know the exact thing you want to run, a bit like keyboard shortcuts in Excel. Both work. Plain English is friendlier while you're learning.

Ask specifically

Name the client, the metric, the window. Be concrete.

  • Weak: "how's Bob doing lately?"
  • Better: "ROAS for BobsWidgetStore, last 14 days vs the 14 before that, GB only"

If you can't be specific because you don't know yet, say that too. "I don't know where to start — which clients in my portfolio dropped hardest this week?" is a perfectly good opener.

Read the evidence, not just the conclusion

Every answer shows you the numbers it used. If something looks off, say so — it'll re-check. If a recommendation feels aggressive, push back.

That ROAS drop was a 3-day blip after a promo ended.
Does the 14-day view still support lowering the target?

It's a junior analyst working for you, not an oracle. Your judgement is the point.

Close the loop

When you apply a change in Admin, tell it. "I just set the target to 2.15 — check it took effect." It'll verify the change is live. The next morning, open a fresh chat (see section 9) and ask "how did that ROAS change on BobsWidgetStore land overnight?" — it'll pull the numbers and tell you.

4.The bidnamic-os folder

When you install Bidnamic OS, a folder called bidnamic-os appears in your home folder (open Finder and hit ⇧⌘H, or look under ~/bidnamic-os). Drop files into it and Brian can read them — that's how you hand him an artefact.

Things you drop in

Anything you want Brian to look at:

  • A screenshot of Admin UI or a Google Ads dashboard
  • A client email (save the message, or paste the body into a text file)
  • A PDF contract, brief, or deck
  • An exported CSV from anywhere
  • A Gong transcript, a photo of a whiteboard, a Slack screenshot

Then just refer to it by name:

Read the screenshot I just dropped in the folder — bobs-widgets-admin.png.
Tell me whether BobsWidgetStore's bid strategy looks right.
Read the email I just saved into the folder. Draft a reply
in a friendly but firm tone — we can't lower the ROAS target below 2.

The best kind of "ask specifically" is handing over the actual artefact.

Things Brian hands back

When Brian builds something — a report, an investigation summary, a branded HTML deliverable — he saves it for you and tells you the path. You don't need to track where it lands; just open the link Brian gives you, or ask him to remind you:

Where did you save that investigation again?

Double-click to open, forward to a client, attach to an email, or keep for the captain's log.

Habit. Treat the bidnamic-os folder as your working desk for inputs. Drop things in, let Brian rummage. Clear it out at the end of the week if it gets messy.

5.When it pauses to check with you

Most of the time you won't see permission prompts — the environment is pre-authorised, so Brian can run queries, read files, and save outputs without asking. You'll just see the work happening.

The pauses you will see are the ones that matter: a skill asking you to confirm a proposed change, an investigation checking whether to act or wait, or an explicit "does this look right?" at a decision point. Those are yours to answer. Read what's on screen, sanity-check it against what you know about the client, and say yes or push back.

If something surprises you — a recommendation that feels aggressive, a dataset choice you didn't expect — ask. "Why did you pick that?" and "Explain that in plain English" are two of the most useful things you can say.

6.Your first day: six things to try

Work through these in order. Each one takes a few minutes. Tick them off as you go.

  1. Open a terminal and run bidnamic-os. It'll connect to your cloud session and drop you into the chat. Type hello and hit enter. Brian will greet you with your portfolio count — something like "Hello, Lewis. There are 15 active clients in your portfolio. What can I help you with today?" When you say "my clients" in any prompt, Brian scopes to your portfolio automatically.
  2. Your first real question: "scan my portfolio and tell me which clients look worst today". Watch what Brian does. If he pauses to check something with you, read it and answer.
  3. A client you know well: "give me a quick spot-check on <your best client>". Read what Brian comes back with. Does it match what you already know? If not, push back and see what he says.
  4. Drop a random screenshot into the bidnamic-os folder in your home folder. Say "read what I just added to the folder and summarise it in one line". Just to prove to yourself file sharing works.
  5. Ask Brian something outside his remit — "book me a dentist appointment", "check my calendar" — just to see how he handles it. (He'll tell you, politely.)
  6. Type /clear to wipe the chat and start fresh. See section 9 for why this matters.

That's it — you're oriented. Everything below is reference you can come back to.

7.The things you'll do most

Five core workflows, plus two more that are worth knowing. For each one, the plain-English prompt comes first. If you prefer the shortcut, it's underneath. Both work.

Morning triage ≈ 4–8 min

Start your day by asking the system who needs attention. It'll scan your portfolio, rank clients by health, investigate the worst ones, and come back with a structured summary grouped by severity (P1 = urgent, P4 = informational).

Scan my portfolio and tell me who needs attention today —
worst first, with reasons and a proposed next step for each.

Shortcut: /client:portfolio-review

See how this usually plays out

Scan my portfolio and tell me who needs attention today.

Pulling health scores for your 23 clients, then investigating the five worst. Back in a few minutes.

[runs queries, ~5 min] Here's what I found. Two P1s needing action today, three P2s to monitor, and one goal that's been stale for 47 days…

Tell me more about the first P1.

BobsWidgetStore — ROAS 14d is 1.92 against a goal of 2.5 (-23%). Spend is steady, clicks are steady, conversion rate has dropped from 3.1% to 2.2% over the same window. No captain's log entries in 9 days. Likely landing-page or product-availability issue. I'd recommend investigating the top product pages before touching bid strategy.

Spot-check on a client ≈ 60–90 sec

Use before a call, or when a single client keeps coming up. You'll get outstanding healthchecks, recent captain's log entries, stale goals, recent calls and emails, and key contacts — a one-screen read of what's happening with them right now.

Give me a quick read on BobsWidgetStore —
what's going on with them this week?

Shortcut: /client:portfolio-review --client_name="BobsWidgetStore"

Dig into a problem ≈ 2–5 min

When portfolio-review flags something, or a client reports an issue you don't understand yet. The system follows the diagnostics playbook — gathers evidence across healthchecks, performance, config, recent changes — and comes back with a root cause and a recommendation labelled Wait, Act, or Escalate.

ROAS on BobsWidgetStore has dropped about 25% in the last week.
Figure out why, and tell me whether to act or wait it out.

Shortcut: /client:investigate --client_name="BobsWidgetStore" --issue="ROAS drop 25% in 7 days"

Generate a monthly report ≈ 3–6 min

Just ask in plain English. Brian picks the right queries, writes the HTML, brands it, and tells you where it saved.

Build me the monthly Shopping performance report for
BobsWidgetStore — March 2026 vs March 2025, GB region.

Same approach for other deliverables you might want:

  • "Pre-pitch feed quality audit for <prospect> — what are the obvious problems with their product feed?"
  • "Feed Optimisation title structure for BobsWidgetStore — which attributes are missing, where are the experiment opportunities?"
  • "Spend and ROAS for BobsWidgetStore the last 90 days, weekly, as an HTML I can forward."

If Brian's first cut isn't what you want — wrong window, wrong segmentation, missing a chart — say so. He'll rebuild it.

Two more worth knowing

Share of Voice planning for a pitch ≈ 3–5 min

Prepping a conversation about market position? Brian can segment a client's catalogue by revenue contribution, pull search term performance per segment, and tell you which terms are worth tracking in Search Term Tracking (and which are waste).

I'm prepping a Share of Voice conversation with BobsWidgetStore.
Segment their catalogue by revenue, then tell me which search terms
are worth tracking and which are waste.

When nothing quite fits ≈ varies

If your question doesn't match any of the above — some weird ad-hoc thing no-one has asked before — just describe what you want and let Brian explore. He'll look at what tables exist, try a few queries, and come back with something.

I want to know which clients in my portfolio have had their
merchant-centre disapproval rate go up in the last month,
and whether that correlates with ROAS drops. We've never asked this before.

8.Plain-English prompts that just work

Bidnamic OS covers more ground than you might expect — performance across every service line, product and feed health, commercial context from HubSpot, competitive tracking, benchmarking, lifecycle, and more. Everything below is a real prompt you can type today. Brian routes to the right playbook and dataset behind the scenes.

Performance, targets, and trends

Show me BobsWidgetStore's spend and ROAS for the last 14 days,
broken down by week.
How is PMax spend trending against standard Shopping for
BobsWidgetStore over the last 30 days? Is the hybrid split
still working?
Which of my clients are furthest below their ROAS goal this week?

Products, feeds, and Merchant Centre

Which products on BobsWidgetStore are disapproved right now,
and what are the top reasons?
Show me the worst-performing products — meaningful spend, zero
or near-zero conversions — and whether any of them are on
policy warnings.
What's the feed fill rate for BobsWidgetStore's key attributes?
Where are the biggest gaps?

Feed Optimisation experiments

Which Feed Optimisation experiments are running for
BobsWidgetStore, and how are they performing?
Any A/B tests ready to accept in my portfolio this week?
Show me products that failed AI generation in the
structured-title experiment on BobsWidgetStore.

Search terms and Share of Voice

Which Shopping search terms are wasting budget on BobsWidgetStore
over the last 30 days?
Am I tracking the right search terms for BobsWidgetStore in
Search Term Tracking, or should we swap some out?
Who's competing against BobsWidgetStore in the Shopping carousel
for "hiking boots size 10"?

Search Ads (DSA and RSA)

Show me the DSA search terms on BobsWidgetStore that are spending
the most with zero conversions — candidates for negatives.
How is BobsWidgetStore's Search Ads budget split across DSA, RSA,
and broad — and does the mix still make sense given recent ROAS?

Bing and cross-channel

Show me BobsWidgetStore's Bing performance against Google Shopping
over the last month. Is the mirroring up to date?

Commercial context from HubSpot

What's the deal stage for BobsWidgetStore in HubSpot, and when
does the contract come up for renewal?
Summarise the last three calls with BobsWidgetStore — who was on,
what did they ask for, and what did we commit to?
Who's the primary contact at BobsWidgetStore and what's their email?

Benchmarking against category

How does BobsWidgetStore's Shopping ROAS compare to others in
their category? Are we above or below the median?

Lifecycle and status

Any lifecycle risks in my portfolio — stuck activations, clients near
auto-disable, anyone about to drop off this month?
What's the current status of BobsWidgetStore — active, paused,
activating, something else — and when did it last change?

One that needs narrowing down

Some prompts are too broad on purpose, and Brian will ask you to narrow them down. That's a good thing — it's saving you a wasted query. Here's what it looks like:

What search terms are wasting budget on BobsWidgetStore?

Two places to look: Shopping search terms (from Google Ads) and Search Ads terms (DSA and RSA). Most of BobsWidgetStore's spend sits on Shopping, so that's probably where the waste is. Want me to start there, or look at both?

Start with Shopping.

Pulling Shopping search terms for the last 30 days, filtering to terms with >£50 spend and ROAS below 1.5…

When an answer surprises you or a recommendation feels off, push back the same way. That back-and-forth is the whole point.

9.Keeping the chat fresh

One task, one chat. Finish a workflow, then clear the conversation before starting the next one. Mixing morning triage with a client investigation and a report in the same chat makes Brian slower, more confused, and more likely to mix up clients.

Rough rule: clear the chat when any of these is true:

  • You've been chatting for more than about 30 minutes
  • You've switched clients
  • You've switched tasks (triage → report → investigation)
  • The answers start feeling vague or drifting

To clear, type:

/clear

That's it. Fresh slate. Nothing is lost — every skill, playbook, dataset, and report is still there. Only the conversation gets wiped.

Quick heuristic. If you're about to say "actually, forget what I said earlier about Client A, now I want to look at Client B" — that's the moment to /clear instead. Short, focused chats give sharper answers.

Picking up where you left off

If you walked away mid-task — interrupted by a meeting, laptop went to sleep, accidentally closed the terminal — type:

/resume

That brings back the previous chat so you don't have to re-explain the context. Use /resume when you want continuity, /clear when you want a fresh slate. If you're not sure, /clear is the safer bet — focused chats beat continuity.

10.When you're stuck

  • Brian's asking for a client_id. Just type the name — he'll match it for you.
  • He picked a weird dataset. Ask "why did you pick that dataset?". You'll learn something, or catch a mistake.
  • An answer feels off. Say so. "Double-check this against the active clients view", or "show me the raw numbers". Scepticism is a feature.
  • You don't understand the query he ran. "Explain that query in plain English."
  • You want a different angle. "Try again, but this time look at the 30-day window."
  • The connection or launcher is misbehaving. Run bidnamic-os upgrade, then bidnamic-os again. That fixes most things.
  • Brian can't help. Some things are genuinely outside Bidnamic OS's reach. Post in #delivery-team on Slack and tag someone from the tech team — they'll either add what's missing or point you at the right place.

11.Glossary

Skill

A slash command (like /client:portfolio-review) that runs a named workflow. A shortcut for when you already know what you want. Plain English does the same thing.

Playbook

The written procedure behind a skill — step-by-step instructions for things like "how to adjust a ROAS target" or "how to onboard a new Shopping client". You rarely read them directly; skills follow them for you.

Dataset

A pre-written, pre-tested SQL query that returns a specific set of data (e.g., "recent performance for this client"). Skills use datasets so the numbers are always consistent.

Report

An HTML deliverable built from one or more datasets plus a narrative template, branded and styled by Brian. He'll tell you the path when he's saved it; you double-click to open and forward.

Healthcheck

An automated check that runs against each client (delivery, tracking, ROAS, spend, product disapprovals, and so on) and returns OK / WARNING / CRITICAL. You respond to WARNING and CRITICAL.

Captain's log

The per-client change log where AMs document config changes, goal updates, and issue responses. The system reads it to understand what's already been tried, so it won't propose something you did last week.

Questions, feedback, or something the system couldn't help with? Post in #delivery-team on Slack and tag someone from the tech team.
