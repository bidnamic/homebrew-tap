1.What this actually is

You'll open a tool called Claude Code to talk to it. What you're really using is Bidnamic OS — an assistant that already knows every playbook the team follows, every client's history, every healthcheck, every SQL query our analysts have written and tested.

It looks things up, investigates, explains, drafts, and proposes. You decide and act. Nothing goes into Admin UI unless you put it there yourself.

If you've poked at a general-purpose chatbot before, this will feel different. Answers here are grounded in real Bidnamic data and real procedures. When it tells you BobsWidgetStore's ROAS is tracking 18% below goal, it's because it just ran the query — not because it's guessing.

2.Four habits that make it work

Two minutes to learn. Saves you hours later. These apply to any AI tool you'll use at Bidnamic, not just this one.

Pick the right job for it

Most of the time you'll just type plain English and let it figure out what you want. That's the expected way in — talk to it like you'd brief a smart junior colleague. Slash commands like /client:portfolio-review exist as shortcuts for when you already know the exact thing you want to run, a bit like keyboard shortcuts in Excel. Both work. Plain English is friendlier while you're learning.

Ask specifically

Name the client, the metric, the window. Be concrete.

  • Weak: "how's Bob doing lately?"
  • Better: "ROAS for BobsWidgetStore, last 14 days vs the 14 before that, GB only"

If you can't be specific because you don't know yet, say that too. "I don't know where to start — which clients in my portfolio dropped hardest this week?" is a perfectly good opener.

Read the evidence, not just the conclusion

Every answer shows you the numbers it used. If something looks off, say so — it'll re-check. If a recommendation feels aggressive, push back.

That ROAS drop was a 3-day blip after a promo ended.
Does the 14-day view still support lowering the target?

It's a junior analyst working for you, not an oracle. Your judgement is the point.

Close the loop

When you apply a change in Admin, tell it. "I just set the target to 2.15 — check it took effect." It'll verify the change is live. The next morning, open a fresh chat (see section 9) and ask "how did that ROAS change on BobsWidgetStore land overnight?" — it'll pull the numbers and tell you.

3.The bidnamic-os folder — works both ways

When you install Bidnamic OS, a folder called bidnamic-os appears in your home folder (open Finder and hit ⇧⌘H). Drop files into it, pull files out. That's how you share things with the assistant and how it hands things back — anything in that folder, Claude can see.

Things you drop in

Anything you want it to look at:

  • A screenshot of Admin UI or a Google Ads dashboard
  • A client email (save the message, or paste the body into a text file)
  • A PDF contract, brief, or deck
  • An exported CSV from anywhere
  • A Gong transcript, a photo of a whiteboard, a Slack screenshot

Then just refer to it by name:

Read the screenshot I just dropped in the folder — bobs-widgets-admin.png.
Tell me whether BobsWidgetStore's bid strategy looks right.
Read the email I just saved into the folder. Draft a reply
in a friendly but firm tone — we can't lower the ROAS target below 2.

The best kind of "ask specifically" is handing over the actual artefact.

Things it hands back

When it builds something — a report, an investigation summary — it can save the output as an HTML file into the same folder. Double-click to open, forward to a client, attach to an email, or keep for the captain's log. If it doesn't offer to save automatically, just ask:

Save that investigation as an HTML file I can send to my manager.

You'll get a filename back — something like investigation-BobsWidgetStore-2026-04-14.html — and it'll be sitting in the bidnamic-os folder a second later.

Habit. Treat the bidnamic-os folder as your working desk, not as a filing cabinet. Drop things in, let Claude rummage, pull things out. Clear it out at the end of the week if it gets messy.

4.When it pauses to check with you

Most of the time you won't see permission prompts — the environment is pre-authorised, so Claude can run queries, read files, and save outputs without asking. You'll just see the work happening.

The pauses you will see are the ones that matter: a skill asking you to confirm a proposed change, an investigation checking whether to act or wait, or an explicit "does this look right?" at a decision point. Those are yours to answer. Read what's on screen, sanity-check it against what you know about the client, and say yes or push back.

If something surprises you — a recommendation that feels aggressive, a dataset choice you didn't expect — ask. "Why did you pick that?" and "Explain that in plain English" are two of the most useful things you can say to it.

5.Your first day: six things to try

Work through these in order. Each one takes a few minutes. Tick them off as you go.

  1. Double-click the Bidnamic OS launcher (installed on your laptop as part of setup). A window will open and connect you. Type hello and hit enter. You should see a response within a second or two.
  2. Your first real question: "scan my portfolio and tell me which clients look worst today". Watch what it does. If it pauses to check something with you, read it and answer.
  3. A client you know well: "give me a quick spot-check on <your best client>". Read what it comes back with. Does it match what you already know? If not, push back and see what it says.
  4. Drop a random screenshot into the bidnamic-os folder in your home folder. Say "read what I just added to the folder and summarise it in one line". Just to prove to yourself file sharing works.
  5. Ask it something it probably can't do — "book me a dentist appointment", "check my calendar" — just to see how it handles it. (It'll tell you, politely.)
  6. Type /clear to wipe the chat and start fresh. See section 9 for why this matters.

That's it — you're oriented. Everything below is reference you can come back to.

6.The things you'll do most

Four core workflows, plus two more that are worth knowing. For each one, the plain-English prompt comes first. If you prefer the shortcut, it's underneath. Both work.

Morning triage ≈ 4–8 min

Start your day by asking the system who needs attention. It'll scan your portfolio, rank clients by health, investigate the worst ones, and come back with a structured summary grouped by severity (P1 = urgent, P4 = informational).

Scan my portfolio and tell me who needs attention today —
worst first, with reasons and a proposed next step for each.

Shortcut: /client:portfolio-review

See how this usually plays out

Scan my portfolio and tell me who needs attention today.

Pulling health scores for your 23 clients, then investigating the five worst. Back in a few minutes.

[runs queries, ~5 min] Here's what I found. Two P1s needing action today, three P2s to monitor, and one goal that's been stale for 47 days…

Tell me more about the first P1.

BobsWidgetStore — ROAS 14d is 1.92 against a goal of 2.5 (-23%). Spend is steady, clicks are steady, conversion rate has dropped from 3.1% to 2.2% over the same window. No captain's log entries in 9 days. Likely landing-page or product-availability issue. I'd recommend investigating the top product pages before touching bid strategy.

Spot-check on a client ≈ 60–90 sec

Use before a call, or when a single client keeps coming up. You'll get outstanding healthchecks, recent captain's log entries, stale goals, recent calls and emails, and key contacts — a one-screen read of what's happening with them right now.

Give me a quick read on BobsWidgetStore —
what's going on with them this week?

Shortcut: /client:portfolio-review give me a quick read on BobsWidgetStore

Dig into a problem ≈ 2–5 min

When portfolio-review flags something, or a client reports an issue you don't understand yet. The system follows the diagnostics playbook — gathers evidence across healthchecks, performance, config, recent changes — and comes back with a root cause and a recommendation labelled Wait, Act, or Escalate.

ROAS on BobsWidgetStore has dropped about 25% in the last week.
Figure out why, and tell me whether to act or wait it out.

Shortcut: /client:investigate BobsWidgetStore has a ROAS drop of 25% in 7 days

Build a branded report ≈ 3–6 min

When you want a client-ready deliverable with Bidnamic branding applied, or you're starting a new report from scratch. The system builds it and saves the HTML into the bidnamic-os folder so you can open it, forward it, or keep it.

Build me the monthly Shopping performance report for
BobsWidgetStore — March 2026 vs March 2025, GB region.
Save it into the folder.

Shortcut: /report:apply-branding describe the report you want to build

Use this when you want to ensure Bidnamic branding, or when you're starting a new report.

Two more worth knowing

Share of Voice planning for a pitch ≈ 3–5 min

Prepping a conversation about market position? The system can segment a client's catalogue by revenue contribution, pull search term performance per segment, and tell you which terms are worth tracking in Search Term Tracking (and which are waste).

I'm prepping a Share of Voice conversation with BobsWidgetStore.
Segment their catalogue by revenue, then tell me which search terms
are worth tracking and which are waste.

When nothing quite fits ≈ varies

If your question doesn't match any of the above — some weird ad-hoc thing no-one has asked before — just describe what you want and let it explore. It'll look at what tables exist, try a few queries, and come back with something. If the question turns out to be a recurring one, it can turn it into a reusable pattern so the next person who asks gets the answer faster.

I want to know which clients in my portfolio have had their
merchant-centre disapproval rate go up in the last month,
and whether that correlates with ROAS drops. We've never asked this before.

One that needs narrowing down

Some prompts are too broad on purpose, and the system will ask you to narrow them down. That's a good thing — it's saving you a wasted query. Here's what it looks like:

What search terms are wasting budget on BobsWidgetStore?

Two places to look: Shopping search terms (from Google Ads) and Search Ads terms (DSA and RSA). Most of BobsWidgetStore's spend sits on Shopping, so that's probably where the waste is. Want me to start there, or look at both?

Start with Shopping.

Pulling Shopping search terms for the last 30 days, filtering to terms with >£50 spend and ROAS below 1.5…

When an answer surprises you or a recommendation feels off, push back the same way. That back-and-forth is the whole point.

8.Keeping the chat fresh

One task, one chat. Finish a workflow, then clear the conversation before starting the next one. Mixing morning triage with a client investigation and a report in the same chat makes the assistant slower, more confused, and more likely to mix up clients.

Rough rule: clear the chat when any of these is true:

  • You've been chatting for more than about 30 minutes
  • You've switched clients
  • You've switched tasks (triage → report → investigation)
  • The answers start feeling vague or drifting

To clear, type:

/clear

That's it. Fresh slate. Nothing is lost — every skill, playbook, dataset, and report is still there. Only the conversation gets wiped.

Quick heuristic. If you're about to say "actually, forget what I said earlier about Client A, now I want to look at Client B" — that's the moment to /clear instead. Short, focused chats give sharper answers.

Saving a conversation to come back to

If you want to save the conversation for future reference, type /rename conversation-name-you-want. The next time, use /resume to find it easily by that name and pick the conversation back up — handy for carrying on with something until the problem is resolved.

9.When you're stuck

  • It's asking for a client_id. Just type the name — it'll match it for you.
  • It picked a weird dataset. Ask "why did you pick that dataset?". You'll learn something, or catch a mistake.
  • An answer feels off. Say so. "Double-check this against the active clients view", or "show me the raw numbers". Scepticism is a feature.
  • You don't understand the query it ran. "Explain that query in plain English."
  • You want a different angle. "Try again, but this time look at the 30-day window."
  • It can't help. Some things are genuinely outside the system's reach. Post in #delivery-team on Slack and tag someone from the tech team — they'll either add what's missing or point you at the right place.

10.Glossary

Skill

A slash command (like /client:portfolio-review) that runs a named workflow. A shortcut for when you already know what you want. Plain English does the same thing.

Playbook

The written procedure behind a skill — step-by-step instructions for things like "how to adjust a ROAS target" or "how to onboard a new Shopping client". You rarely read them directly; skills follow them for you.

Dataset

A pre-written, pre-tested SQL query that returns a specific set of data (e.g., "recent performance for this client"). Skills use datasets so the numbers are always consistent.

Healthcheck

An automated check that runs against each client (delivery, tracking, ROAS, spend, product disapprovals, and so on) and returns OK / WARNING / CRITICAL. You respond to WARNING and CRITICAL.

Captain's log

The per-client change log where AMs document config changes, goal updates, and issue responses. The system reads it to understand what's already been tried, so it won't propose something you did last week.
